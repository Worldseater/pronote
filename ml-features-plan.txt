Pronote — план интеграции простой обучаемой модели
Дата фиксации идей: 2026-06-04
Статус: к реализации (обсуждение и старт — завтра)

================================================================================
ОБЩИЙ ПОДХОД
================================================================================

- Не большая LLM «для всего», а маленькие специализированные модели / классификаторы.
- Обучение на действиях пользователя: модель предлагает → пользователь принимает или
  исправляет → исправление сохраняется как обучающий пример (localStorage, позже БД).
- UI: ненавязчивые подсказки с кнопками «Принять» / «Изменить».
- Возможная структура кода (черновик):

  js/ml/
    features.js         — признаки из текста задачи/заметки
    task-classifier.js  — раздел, подраздел, проект, ответственный
    similarity.js       — похожие задачи и заметки
    note-to-task.js     — черновик задачи из заметки
    training-store.js   — примеры для обучения
    train.js            — переобучение (по кнопке или автоматически)

================================================================================
1. КЛАССИФИКАТОР ЗАДАЧ (приоритет — первый старт)
================================================================================

Когда: при создании или редактировании задачи (модалка item-edit-modal).

Вход: название (title) + описание (text).

Модель предлагает:

  1) Раздел (доска):
     - Идеи
     - Разработка
     - Срочно

  2) Подраздел (секция внутри доски):

     Идеи:
       - Новые
       - Обсуждаются
       - Обсудились
       - В будущем

     Разработка:
       - Планируются
       - Делаются
       - Отложены
       - Ожидание

     Срочно:
       - (уточнить при реализации — одна колонка или свои секции)

  3) Ответственного (assigneeId из pronote.assignees.v1)

  4) Проект (projectId из pronote.projects.v1)

Обучение:
  - Сохранять пары: { title, text, prediction, userChoice, timestamp }
  - Каждое «Изменить» / ручное исправление после подсказки = новый пример
  - Переобучение маленькой сети (Brain.js / простой MLP) или правила + классификатор

UI (черновик):
  «Предложено: Разработка → Планируются · Проект X · Иван»
  [Принять]  [Изменить]

================================================================================
2. ПОХОЖАЯ ЗАДАЧА / ЗАМЕТКА УЖЕ ЕСТЬ
================================================================================

Когда: при создании новой задачи или заметки (до сохранения или сразу после ввода
title/текста).

Действие:
  - Анализ всего списка задач (Идеи + Разработка + Срочно + Готово?) и заметок
  - Если найдено похожее — показать предупреждение со ссылкой/превью

Пример UI:
  «Похоже на задачу „Настроить деплой“ (Разработка, 2 недели назад)»
  «Похоже на заметку „Идеи по онбордингу“»

Техника (этапами):
  - MVP: TF-IDF / косинусное сходство по title + text
  - Позже: эмбеддинги или маленькая сеть для ранжирования похожести

Порог похожести — настраиваемый (обсудить завтра).

================================================================================
3. ЗАМЕТКА → ЧЕРНОВИК ЗАДАЧИ
================================================================================

Когда: на странице заметок или в модалке заметки — кнопка «Создать задачу из заметки».

Действие:
  - По тексту заметки сформировать черновик задачи:
    · title — краткий, адекватный заголовок
    · text / описание — структурированное описание
    · раздел + подраздел (п.1)
    · проект (п.1)
    · ответственный (п.1)

  - Открыть модалку создания задачи с предзаполненными полями
  - Пользователь проверяет и сохраняет

Обучение:
  - Если пользователь меняет сгенерированные поля перед сохранением — логировать
    как пример для улучшения формулировок и классификатора из п.1

================================================================================
ПОРЯДОК РЕАЛИЗАЦИИ (предложение на завтра)
================================================================================

  1. Классификатор задач (раздел + подраздел + проект + ответственный)
  2. Похожие задачи / заметки
  3. Заметка → черновик задачи

================================================================================
ВОПРОСЫ НА ЗАВТРА
================================================================================

  - Срочно: есть ли подразделы или одна секция?
  - Искать похожие в «Готово» или только в активных?
  - Минимальная длина текста для срабатывания подсказок?
  - Переобучение: вручную по кнопке в настройках или автоматически?
  - Нужен ли экспорт датасета обучения в JSON для будущего сервера?
